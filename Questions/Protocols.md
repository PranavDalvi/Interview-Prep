## 1. HTTP (Hyper Text Transport Protocol)
### Key Characteristics
- Simple
- Human readable
- Supported by all browsers
- Stateless
### Common Use Cases
- Web browsers
***
## 2. WebSockets
### Key Characteristics
- Bi-directional Communication
- Persistent Connection
- low-latency
- stateful
### Common use cases
- Chat Application
- Live dashboards
- Collaborative editing
***
## 3. Server-Sent Events
### Key Characteristics
- One-way communication (server to client)
- Human Readable
### Common use cases
- News feeds
- status updates
- stock tickers
***
## 4. gRPC (Google Remote Procedure Call)
### Key Characteristics
- Binary protocol (HTTP/2)
- Strongly-typed contracts (protocol buffers)
- Requires code generation
### Common use cases
- Microservice communication
- Performance critical systems
- IoT devices
***
## 5. REST (REpresentational State Transfer)
### Key Characteristics
- Multiple endpoints
- Human readable
- Supported by all browsers
- Stateless
### Common use cases
- Single-source of data
- CRUD apps
- easily cached data
***
## 6. GraphQL (Graph Query Language)
### Key Characteristics
- Single endpoint
- Precise data retrieval
- Self-documenting API
- Strongly typed
### Common use cases
- Complex or multiple sources of data (Facebook Homepage)
- Apps supporting multiple client types
- Decoupling frontend from backend development
